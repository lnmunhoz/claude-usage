# Claude Code Instructions

## GitHub Issues

When you are triggered by a GitHub issue (not a PR comment), you MUST:

1. Create a new branch from `main` with a descriptive name based on the issue
2. Implement the requested changes
3. Commit the changes
4. Push the branch and create a Pull Request that references the issue (use "Closes #N" in the PR body)

Do NOT just comment on the issue with suggestions. Always create a PR with the implementation.
